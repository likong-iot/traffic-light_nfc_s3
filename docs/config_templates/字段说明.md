# 配置字段说明

设备配置文件为 JSON 格式，正式文件名为 SD 卡根目录的 `CONFIG.JSN`。

## 顶层字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `version` | 数字 | 配置格式版本。当前为 `2`。 |
| `schedule` | 对象 | 继电器 1、继电器 2 的定时闭合配置。 |
| `nfc` | 对象 | NFC 卡数据和动作映射。 |
| `radar` | 对象 | 外部雷达输入触发配置。 |

## `schedule`

```json
"schedule": {
  "relay1": {"start": "18:00", "end": "07:00"},
  "relay2": {"start": "18:00", "end": "07:00"}
}
```

| 字段 | 说明 |
| --- | --- |
| `relay1.start` | 继电器 1 闭合开始时间。 |
| `relay1.end` | 继电器 1 闭合结束时间。 |
| `relay2.start` | 继电器 2 闭合开始时间。 |
| `relay2.end` | 继电器 2 闭合结束时间。 |

时间格式必须是 `HH:MM`，例如 `18:00`。支持跨天时间段，例如 `18:00` 到 `07:00` 表示晚上 6 点闭合，第二天早上 7 点断开。

## `nfc.rules`

```json
"nfc": {
  "rules": [
    {"data": "00", "name": "1类卡", "opto12_pulses": 1, "open_relay4": false}
  ]
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data` | 字符串 | 匹配卡内 `data[0..1]`，两位十六进制字符，例如 `00`、`01`、`0A`。 |
| `name` | 字符串 | 规则名称，用于网页显示和日志识别。 |
| `opto12_pulses` | 数字 | `OPTO1 + OPTO2` 同时输出的脉冲个数。 |
| `open_relay4` | 布尔值 | 是否断开继电器 4。继电器 4 开机默认闭合，`true` 表示触发后断开。 |

固件只按卡内 `data[0..1]` 匹配，不按 UID 匹配。

## `radar`

```json
"radar": {
  "enabled": true,
  "active_level": 1,
  "trigger_delay_ms": 5000,
  "cycle_window_ms": 20000,
  "opto12_pulses": 1
}
```

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `enabled` | 布尔值 | `true` | 是否启用雷达触发。 |
| `active_level` | 数字 | `1` | 雷达 INT 有效电平。新版 PCB 为高电平有效。 |
| `trigger_delay_ms` | 数字 | `5000` | 收到雷达触发后延时多久输出脉冲。 |
| `cycle_window_ms` | 数字 | `20000` | 周期窗口。窗口内重复触发只输出一次。 |
| `opto12_pulses` | 数字 | `1` | 雷达触发后 `OPTO1 + OPTO2` 输出脉冲个数。 |

`IO_IN1/GPIO16` 和 `IO_IN2/GPIO38` 共用同一个周期窗口。
