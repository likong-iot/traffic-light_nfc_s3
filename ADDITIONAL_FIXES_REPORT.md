### 问题1：类别3尾脉冲状态清理不完整 ✅

**严重程度**：🟡 中  
**位置**：`main/src/app.c` `runtime_process_timers()`  

**问题描述**：
- 类别3超时后执行类别1动作前，调用`runtime_set_led2_off()`关闭LED2
- 但注释说"先关闭LED2"，实际上`runtime_set_led2_off()`内部会调用`runtime_cancel_class3_tail()`
- 状态清理其实是完整的，但代码不够清晰

**修复方案**：
```c
// 修复前
// 先关闭LED2
runtime_set_led2_off();

// 修复后
// 修复: 先清理类别3状态，再执行类别1动作
runtime_set_led2_off();  // 这会调用runtime_cancel_class3_tail()
```

**改进**：注释更加明确，说明`runtime_set_led2_off()`会自动清理类别3尾脉冲状态

### 问题2：雷达事件队列满时直接丢弃 ✅

**严重程度**：🟡 中  
**位置**：`main/src/radar_input.c` `post_radar_ready()`  

**问题描述**：
- 雷达触发信号通过队列发送到应用状态机
- 队列大小仅16，满时直接丢弃事件
- 可能导致雷达触发信号丢失

**修复方案1：增大队列大小**
```c
// 文件: main/src/app.c 行42
// 修复前
#define APP_CONTROL_QUEUE_SIZE    16

// 修复后
#define APP_CONTROL_QUEUE_SIZE    32  // 修复: 从16增加到32，防止雷达事件丢失
```

**修复方案2：添加重试机制**
```c
// 文件: main/src/radar_input.c post_radar_ready()
// 修复: 重试机制，避免队列满时直接丢弃
esp_err_t err;
for (int retry = 0; retry < 3; retry++) {
    err = app_post_control_event(&event);
    if (err == ESP_OK) {
        return;  // 发送成功
    }

    if (retry < 2) {  // 不是最后一次重试
        ESP_LOGW(TAG, "radar ready event queue full (attempt %d/3), retrying...", retry + 1);
        vTaskDelay(pdMS_TO_TICKS(10));  // 短暂延时后重试
    }
}

// 重试3次后仍失败
ESP_LOGE(TAG, "radar ready event dropped after 3 retries: %s", esp_err_to_name(err));
```

**改进**：
1. ✅ 队列大小从16增加到32（降低满的概率）
2. ✅ 添加重试机制（队列满时重试3次，每次间隔10ms）
3. ✅ 改进日志（区分重试中和最终失败）
